# Capítulo 3: Estrutura Básica dos Prompts

Este capítulo trata prompt como artefato de engenharia. Ele é importante porque mostra como reduzir ambiguidade, separar regras permanentes de pedidos locais e transformar instruções em contratos mais previsíveis.

## Estrutura da pasta

| Caminho | Papel |
| --- | --- |
| `estrutura-basica-dos-prompts.md` | aula principal do capítulo |
| `history.md` | decisões editoriais, contexto removido e memória de reorganização |
| `examples/` | exemplos práticos do conteúdo |
| `AGENTS.md` | orientações curtas para agentes |

## O que estudar aqui

O capítulo aprofunda:

- ambiguidade como causa de saídas ruins;
- escopo e limites explícitos;
- trade-offs entre custo, latência e qualidade;
- zero-shot vs few-shot;
- separação entre `System Prompt` e `User Prompt`;
- estrutura base de prompts robustos;
- formatos de saída, critérios de qualidade e tratamento de erro.

Para a explicação completa, leia `estrutura-basica-dos-prompts.md`.

## Como usar os exemplos

Os exemplos desta pasta ajudam a visualizar o conteúdo da aula, sem substituir a leitura do material principal. Eles foram modelados para mostrar comparações reais de prompt, em vez de apenas imprimir textos estáticos.

## Importância do capítulo

Este é o ponto em que o curso sai de técnicas isoladas e passa a discutir design de instruções com rigor de software. Ele prepara o terreno para workflows, versionamento e avaliação.
